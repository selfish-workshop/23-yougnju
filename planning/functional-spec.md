# 기능 명세서 (Functional Specification)

**작성**: 기획팀
**참조 문서**: `/docs/superpowers/specs/2026-04-10-사라지는-메모장-design.md` (PM 스펙)
**문서 범위**: PM이 결정한 기능들을 기능 단위로 쪼개 각각의 동작을 구체화

---

## 0. 문서 읽는 법

- 각 기능은 **FN-##** 형식의 고유 ID를 가진다
- "트리거"는 기능이 실행되는 조건, "동작"은 실행 시 일어나는 일, "결과 상태"는 실행 후 상태
- 우선순위(P0/P1/P2)는 `mvp-priority.md` 참조
- 이 문서는 "무엇을, 어떻게 동작시킬지"만 정의한다. 시각적 디테일(색상·폰트·모션 수치)은 디자인팀 영역

## FN-01. 첫 진입 안내

| 항목 | 내용 |
|------|------|
| **목적** | 첫 방문자에게 서비스 사용법을 1회만 알림 |
| **트리거** | URL 접속 시 `localStorage.hasSeenIntro !== true` |
| **동작** | 캔버스 상단 또는 중앙에 한 줄 안내 문구 노출: "적어봐요. 5분 뒤 사라집니다." (카피는 CX팀이 최종 확정) |
| **종료 조건** | ① 사용자가 화면 탭 ② 3초 경과 ③ 타이핑 시작 중 하나 발생 |
| **후속 동작** | `localStorage.hasSeenIntro = true` 저장 → 다음 방문부터 노출 안 됨 |
| **비기능** | 안내 노출 중에도 캔버스는 이미 활성화 상태(즉시 타이핑 가능해야 함) |

---

## FN-02. 캔버스 타이핑

| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 자유롭게 생각을 쓸 수 있는 기본 입력 영역 |
| **트리거** | 캔버스 영역 탭 또는 페이지 진입 후 자동 포커스 |
| **동작** | 텍스트 에디터 영역 활성화. 멀티라인 입력 지원. 제목 없음, 본문만 |
| **입력 범위** | 한국어·영어·이모지·특수문자 모두 허용 |
| **길이 제한** | 없음 (단, 10,000자 초과 시 경고 없이 그대로 진행) |
| **자동 저장** | **없음** (localStorage에도 절대 저장 안 함) |
| **스크롤** | 내용이 화면 높이를 초과하면 캔버스 내부 스크롤 발생 |
| **관련 기능** | FN-03(타이머 시작), FN-04(햅틱) |

---

## FN-03. 타이머 시작

| 항목 | 내용 |
|------|------|
| **목적** | 첫 글자 입력 순간부터 카운트다운 시작 |
| **트리거** | 캔버스에 첫 문자가 입력된 순간 (공백·개행 제외, 실제 글자) |
| **동작** | 현재 슬라이더 값(초)을 타이머로 설정 → 카운트다운 시작 |
| **시각 피드백** | 상단 타이머 UI에 남은 시간 표시. 1초 단위 갱신 |
| **추가 타이핑** | 이미 카운트다운 중이면 **타이머에 영향 없음** (첫 입력 기준 유지) |
| **중복 방지** | 타이머는 한 세션당 한 번만 시작. 재시작 불가 |

---

## FN-04. 햅틱 피드백

| 항목 | 내용 |
|------|------|
| **목적** | 밤에 소리 없이도 감각적 피드백 제공 |
| **트리거 1: 타이핑** | 첫 글자 입력 순간, 미묘한 짧은 진동 (약 10ms) |
| **트리거 2: 사라짐 직전** | 잉크 번짐 애니메이션 시작 직전, 부드러운 중간 진동 (약 30ms) |
| **트리거 3: 퇴장 안내 종료** | "자러 가볼까요?" 노출 종료 시 가장 긴 진동 (약 50ms) |
| **API** | `navigator.vibrate()` 사용 |
| **미지원 시** | 조용히 무시 (에러 표시 안 함) |
| **설정** | `localStorage.hapticEnabled = false` 시 모든 진동 비활성화 |

## FN-05. 타이머 슬라이더

| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 머무를 시간을 선택 |
| **위치** | 화면 상단 고정 |
| **기본 상태** | 접힌 상태(현재 설정 값만 표시, 예: "⏱ 5분") |
| **상호작용** | 탭하면 펼쳐져서 슬라이더 노출 → 드래그하여 시간 조정 → 바깥 탭하면 접힘 |
| **범위** | 30초 ~ 1시간 (30초, 1분, 3분, 5분, 10분, 15분, 30분, 1시간 눈금) |
| **기본값** | `localStorage.preferredDuration` 있으면 사용, 없으면 300초(5분) |
| **조정 (입력 전)** | 실시간으로 선택 값만 갱신, 타이머는 아직 시작 전 |
| **조정 (입력 후)** | 남은 시간을 새 설정값 비율로 재계산, 조용히 갱신 (확인 프롬프트 없음) |
| **저장** | 슬라이더에서 손 떼면 `localStorage.preferredDuration`에 즉시 저장 |

**재계산 공식** (입력 중 조정 시):
```
newRemaining = currentRemaining × (newDuration ÷ previousDuration)
```
예: 5분 설정 → 2분 남은 상태에서 10분으로 변경 → 새 남은 시간 = 2 × (600/300) = 4분

## FN-06. 잉크 번짐 애니메이션

| 항목 | 내용 |
|------|------|
| **목적** | 타이머 종료 시 메모가 "사라지는" 시그니처 모먼트 |
| **트리거** | 타이머 0초 도달 |
| **동작** | 캔버스 전체 글자가 잉크가 물에 번지듯 서서히 흐려지고 확산 → 완전 투명 |
| **지속 시간** | 2~3초 (정확한 수치는 디자인팀이 결정) |
| **중간 개입** | **불가**. 애니메이션 시작 후엔 취소·빠르게 감기 등 일체 없음 |
| **끝난 뒤 상태** | 캔버스 내용은 DOM에서 완전 제거 (복구 불가능하게) |
| **구현 힌트** | CSS `filter: blur()`과 `opacity` 조합으로 가능. 디자인팀과 협의 |
| **접근성** | `prefers-reduced-motion: reduce` 감지 시 단순 페이드아웃으로 대체 |

---

## FN-07. 퇴장 안내

| 항목 | 내용 |
|------|------|
| **목적** | 연속 사용 유혹을 줄이고, "이제 자러 갈 시간"을 암시 |
| **트리거** | FN-06(잉크 번짐) 완료 직후 |
| **동작** | 빈 화면 중앙에 문구 노출: "오늘은 여기까지. 이제 자러 가볼까요?" (카피는 CX팀이 최종 확정) |
| **지속 시간** | 3초 고정 |
| **동안 사용자 행동** | ① 화면 탭 → 새 빈 캔버스로 즉시 이동 ② 3초 경과 → FN-08 자동 전환 |
| **강제성** | 연속 사용을 막지 않음 (탭하면 바로 새 캔버스) |

---

## FN-08. 새 빈 캔버스 재시작

| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 원하면 다시 쓸 수 있도록 초기화 |
| **트리거** | ① FN-07 퇴장 안내 종료 ② 사용자가 사라진 상태에서 화면 탭 |
| **동작** | 캔버스를 빈 상태로 리셋, 타이머는 아직 시작 전(슬라이더 값만 유지) |
| **유지되는 것** | 슬라이더 설정값, 햅틱 설정, 첫 진입 플래그 (localStorage 그대로) |
| **초기화되는 것** | 캔버스 내용 (이미 비어 있음), 타이머 상태 |

## FN-09. 설정 저장 (localStorage)

| 항목 | 내용 |
|------|------|
| **목적** | 재방문 시 사용자 편의 유지 |
| **저장 시점** | 관련 이벤트 발생 즉시 |
| **저장 항목** | `preferredDuration`, `hasSeenIntro`, `hapticEnabled` |
| **절대 저장 안 하는 항목** | 캔버스 내용, 사용자 식별자, IP, 접속 시각, 사용 횟수 |
| **접근 실패 시** | localStorage 미지원·프라이빗 모드 등 → 세션 기본값으로 동작, 매번 초기화 |
| **검증** | 각 값의 형식·범위 체크. 불량 데이터면 기본값으로 덮어씀 |

**스키마**:
```js
{
  "preferredDuration": 300,    // 30 ~ 3600 (초)
  "hasSeenIntro": true,        // boolean
  "hapticEnabled": true        // boolean
}
```

## 기능 간 의존 관계 요약

```
FN-02 (캔버스 타이핑)
  └─ 첫 입력 → FN-03 (타이머 시작)
                   └─ FN-04 (타이핑 햅틱)
                   └─ 카운트다운 종료 → FN-06 (잉크 번짐)
                                            └─ FN-04 (사라짐 햅틱)
                                            └─ 종료 → FN-07 (퇴장 안내)
                                                         └─ 3초 또는 탭 → FN-08 (새 캔버스)
                                                                              └─ FN-02로 순환

FN-05 (슬라이더) ↔ FN-03 (타이머 시작·재계산)
FN-09 (localStorage) ← FN-05, FN-01(안내 플래그), FN-04(햅틱 설정)
FN-10 (다크 테마) — 전 구간 공통
FN-11 (반응형) — 전 구간 공통
FN-01 (첫 진입 안내) — FN-02 직전에만 실행
```

## 향후 확장 시 관계 (MVP 외)

- **BGM 토글**: FN-04와 같은 설정 범주. localStorage에 `bgmEnabled` 추가
- **라이트 테마**: FN-10 확장. 시스템 테마 감지 또는 수동 토글
- **공유 카드**: 새 기능군. 사라진 글은 공유 불가이므로 "사용 경험 자체"를 공유하는 카드
