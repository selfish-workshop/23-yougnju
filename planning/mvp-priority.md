# MVP 우선순위 (Priority Matrix)

**작성**: 기획팀
**참조**: PM 스펙 9장(MVP 범위) + functional-spec.md
**범위**: PM이 "MVP 포함"으로 결정한 모든 기능을 P0/P1/P2 3단계로 분류하고 순서를 매긴다

---

## 0. 우선순위 레벨 정의

| 레벨 | 의미 | 개발 순서 |
|------|------|----------|
| **P0** | 이게 없으면 서비스 자체가 성립 안 됨. **Day 1 런칭 필수** | 가장 먼저 |
| **P1** | 브랜드 경험의 핵심. 일정이 촉박하면 축소판으로라도 꼭 포함 | 두 번째 |
| **P2** | 있으면 좋지만, 밀려도 런칭은 가능. 런칭 후 첫 주에 추가 가능 | 마지막 |

⚠️ **주의**: P0/P1/P2 모두 PM의 MVP 범위 안이다. "MVP 제외"는 따로 있음(PM 스펙 9장 참조)

---

## 1. P0 — Day 1 필수 (이게 빠지면 서비스가 성립 안 됨)

| FN | 기능 | 이유 |
|----|------|------|
| **FN-02** | 캔버스 자유 타이핑 | 서비스의 본질. 없으면 앱 자체가 없음 |
| **FN-03** | 타이머 시작 (첫 입력 기준) | "5분 뒤 사라진다"의 핵심 메커니즘 |
| **FN-06** | 잉크 번짐 사라짐 (또는 최소 페이드아웃) | 사라지지 않으면 거짓말. 단 애니메이션이 부족하면 페이드아웃으로 임시 대체 가능 |
| **FN-11** | 모바일 반응형 (세로 모드) | 타겟이 모바일이므로 데스크탑 없이도 런칭 가능. 역은 불가 |
| **FN-10** | 다크 테마 | 유일한 테마이므로 자동 포함 |

**P0만 있는 상태**: 사용자가 URL 접속 → 빈 캔버스 → 타이핑 → 5분 뒤 사라짐. 끝.
→ **이것만으로도 서비스 정체성은 성립.** 단지 브랜드 경험의 깊이가 없음

---

## 2. P1 — 브랜드 경험 핵심 (강력히 포함)

| FN | 기능 | P1인 이유 |
|----|------|----------|
| **FN-05** | 타이머 슬라이더 | PM이 "5분 고정"이 아닌 슬라이더를 명시적으로 선택. 이게 없으면 "5분짜리 메모장"으로 축소됨 |
| **FN-07** | "자러 가볼까요?" 퇴장 안내 | PM이 강조한 브랜드 포지셔닝 ("불면증 직장인의 새벽 친구"). 이게 없으면 그냥 "사라지는 메모장" |
| **FN-09** | localStorage 설정 저장 | 슬라이더가 있는 이상 기억 기능도 필수. 재방문 시 매번 초기화되면 부자연스러움 |
| **FN-01** | 첫 진입 안내 | 서비스 컨셉 자체가 생소하므로, 첫 방문자에게 1초라도 설명이 필요 |
| **FN-08** | 새 빈 캔버스 재시작 | 연속 사용 가능해야 자연스러움 |

**P0+P1 상태**: 사용자 여정의 감정 루프 전체가 작동. 브랜드 경험 완성.

---

## 3. P2 — 런칭 후 첫 주 가능 (밀릴 수 있음)

| FN | 기능 | P2인 이유 |
|----|------|----------|
| **FN-04** | 햅틱 피드백 | 기기 의존적(iOS Safari 제한적, 일부 안드로이드만). 없어도 시각+글자만으로 체험 가능. 추가는 빠르게 가능 |
| **FN-11 확장** | 데스크탑 중앙 정렬 | 모바일 우선 전략. 데스크탑은 모바일 뷰포트로 표시되어도 무방 |

**P0+P1+P2 상태**: 스펙상의 모든 기능이 동작. MVP 완료.

## 4. 개발 순서 로드맵

### 스프린트 1 (P0 Only) — "작동하는 프로토타입"
```
1. HTML/CSS 기본 레이아웃 (다크 테마, 모바일)
2. 캔버스 타이핑 동작 (contenteditable 또는 textarea)
3. 첫 입력 감지 → 5분 타이머 (하드코딩)
4. 타이머 종료 → 단순 페이드아웃 → 빈 화면
```
**완료 기준**: 모바일 브라우저에서 URL 접속 → 타이핑 → 5분 뒤 사라짐

### 스프린트 2 (P1 추가) — "브랜드 경험 완성"
```
5. 상단 슬라이더 UI (접힘/펼침)
6. 슬라이더 ↔ 타이머 연동 (입력 전·중 모두)
7. localStorage 읽기/쓰기 (preferredDuration, hasSeenIntro)
8. 첫 진입 안내 1회 노출
9. 잉크 번짐 애니메이션 (CSS blur+opacity)
10. "자러 가볼까요?" 안내 + 탭 처리 + 새 캔버스
```
**완료 기준**: PM 스펙의 사용자 여정 전체가 경험됨

### 스프린트 3 (P2 추가) — "마감 작업"
```
11. 햅틱 피드백 3지점(타이핑·사라짐·퇴장 종료)
12. 데스크탑 중앙 정렬 반응형
13. 접근성 (키보드 네비게이션, prefers-reduced-motion)
14. 런칭 전 QA 체크리스트 돌기
```
**완료 기준**: MVP 전체 완성, 런칭 준비

## 5. 일정 단축 시나리오 — "2주 안에 런칭해야 함"

만약 개발 리소스가 부족해 2주 안에 런칭해야 한다면:

| 스프린트 | 기간 | 내용 |
|---------|------|------|
| 1주차 | P0 전부 완성 | 최소 동작 프로토타입 |
| 2주차 전반 | P1 중 FN-05(슬라이더), FN-07(퇴장 안내), FN-09(localStorage) | 브랜드 경험 최소 완성 |
| 2주차 후반 | 디버그·QA·런칭 | 배포 |

**희생**: FN-01(첫 진입 안내), FN-04(햅틱), FN-08(명시적 재시작), 데스크탑 정렬

**런칭 후 첫 주**: 희생한 기능들 바로 추가 반영

```
영향도   | FN-02 캔버스 ⭐           | FN-06 잉크 번짐 ⭐
  높음   | FN-03 타이머 시작 ⭐      | FN-05 슬라이더 ⭐
         | FN-10 다크 테마 ⭐        |
         | FN-07 퇴장 안내 ⭐        |
---------|--------------------------|------------------------
영향도   | FN-09 localStorage       | FN-04 햅틱
  보통   | FN-08 새 캔버스           | FN-11 데스크탑 반응형
         | FN-01 첫 진입 안내        |
```

⭐ = P0 또는 P1 핵심

## 7. 의사결정 필요 항목 (PM에게 요청)

🚨 **다음 항목은 기획팀이 확정할 수 없음. PM 결정 필요**:

### (a) 런칭 일정 — 2주? 3주? 4주?
일정에 따라 P2를 MVP에 포함할지 여부가 갈림

### (b) 백그라운드 타이머 동작 (user-scenarios.md 시나리오 5 참조)
사용자가 폰을 잠그거나 탭을 나가도 타이머가 계속 도는가?
- **옵션 A**: 계속 돔 (JavaScript setTimeout 활용, 모바일 지연 가능)
- **옵션 B**: 포그라운드 복귀 시 현재 시각 기준 재계산 (더 정확하지만 구현 복잡)

### (c) 접근성 수준
WCAG AA를 MVP에서 맞출지, 런칭 후 개선할지
- **옵션 A**: P0에 포함 (색 대비, 키보드 네비)
- **옵션 B**: P2로 미룸
