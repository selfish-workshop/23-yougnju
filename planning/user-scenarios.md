# 사용자 시나리오 (User Scenarios)

**작성**: 기획팀
**참조**: PM 스펙 2장(타겟 사용자) + functional-spec.md
**범위**: 대표 페르소나 "새벽 3시 잠 못 드는 직장인"을 중심으로 주요 시나리오 5가지 정리

---

## 0. 시나리오 작성 원칙

- 각 시나리오는 **5W1H**(Who/When/Where/Why/What/How) 구조
- 시나리오는 **FN-## 기능 호출**을 명시해서 기획-명세 매핑 가능하게 함
- 감정 상태도 함께 기록(이 서비스의 본질이 감정 제품이기 때문)

---

## 시나리오 1 — 첫 방문 (새벽 2:47am)

| 항목 | 내용 |
|------|------|
| **Who** | 30대 초 직장인, 내일 중요한 프레젠테이션이 있음 |
| **When** | 새벽 2:47, 침대에서 1시간째 뒤척임 |
| **Where** | 집 침실, 폰 한 손에 들고 누움 |
| **Why** | 머릿속에 "내일 발표 망하면 어쩌지", "아까 팀장 말투가 이상했어" 같은 생각이 꼬리 무는 중. 어디다 쏟아놓고 싶은데 SNS는 안 되고, 메모장에 쓰기도 부담스러움 |
| **What** | 우연히 트위터에서 "사라지는 메모장"이라는 서비스를 보고 링크 클릭 |
| **How** | 브라우저에서 URL 접속 → 빈 캔버스와 짧은 안내 "적어봐요. 5분 뒤 사라집니다." 노출 → 3초 뒤 사라짐 → 바로 타이핑 시작 |

**거치는 기능**: FN-01 (첫 진입 안내) → FN-02 (캔버스 타이핑) → FN-03 (타이머 시작, 기본 5분) → FN-04 (타이핑 햅틱)

**감정 변화**:
- 진입 전: 불안·짜증
- 안내 보는 순간: "어? 진짜 사라져?" 호기심
- 타이핑 중: 처음엔 망설임 → 점점 쏟아내는 느낌, 안도감
- 타이머 종료 직후: "진짜 사라졌네" 허탈감 + 후련함 공존

**성공 기준**: 이 사용자가 첫 방문 후 **URL을 즐겨찾기에 추가**하거나, 다음 날 밤 다시 접속함

## 시나리오 2 — 반복 사용 (일주일 뒤)

| 항목 | 내용 |
|------|------|
| **Who** | 시나리오 1과 동일 사용자 |
| **When** | 일주일 후, 새벽 1:30 |
| **Where** | 똑같이 침실 |
| **Why** | 요즘 회사 일이 계속 많아서 머리가 복잡함. 이제 이 서비스가 "내 새벽 친구"로 자리 잡음 |
| **What** | 즐겨찾기에서 바로 접속 |
| **How** | URL 접속 → 빈 캔버스(첫 진입 안내 없음, 이미 봤음) → 슬라이더에 "⏱ 5분" 표시됨(지난번 설정 기억) → 바로 타이핑 |

**거치는 기능**: FN-02 → FN-03 → FN-04 → FN-06 → FN-07 → FN-08 → (필요 시) FN-02 다시

**감정 변화**:
- 진입 전: 피로·막막함
- 캔버스 대면: "여기 왔구나" 익숙한 안정감
- 타이핑 중: 일주일 치 축적된 감정을 털어냄
- 퇴장 안내 "이제 자러 가볼까요?": 마치 친구가 다독이는 느낌

**성공 기준**: 한 번의 세션으로 **실제로 졸림을 느끼거나 폰을 내려놓음**

## 시나리오 3 — 타이머 조정 (잠들 수 없는 밤)

| 항목 | 내용 |
|------|------|
| **Who** | 평소 사용자이지만 오늘은 특히 마음이 복잡한 날 |
| **When** | 새벽 3:15, 평소보다 더 길게 머물고 싶음 |
| **Where** | 거실 소파, 오늘은 침대로 가기 전부터 먼저 열어둠 |
| **Why** | 5분으론 부족할 것 같음. 오늘은 오래 쓰고 싶음 |
| **What** | 슬라이더 조정해서 30분으로 설정 |
| **How** | 캔버스 진입 → 상단 "⏱ 5분" 탭 → 슬라이더 펼쳐짐 → 30분 눈금으로 드래그 → 바깥 탭 → 슬라이더 접힘 → 타이핑 시작 |

**거치는 기능**: FN-05 (슬라이더 조정) → localStorage 즉시 갱신 → FN-02 → FN-03 (타이머 30분으로 시작)

**주의점**: 다음 방문 시에도 슬라이더는 **30분 기본값**이 됨(기억됨). 만약 일시적 조정이었다면 다음에 다시 5분으로 바꿔야 함 — 이 점을 사용자가 이해할 수 있도록 CX팀에서 설명 카피 고민 필요

**감정 변화**: "오늘은 오래 써도 돼" 허락받은 느낌

## 시나리오 4 — 중간에 슬라이더 바꾸기

| 항목 | 내용 |
|------|------|
| **Who** | 시나리오 1 사용자 |
| **When** | 5분 설정으로 시작했는데 3분 남은 시점 |
| **Where** | 침대 |
| **Why** | "5분은 너무 짧네, 조금만 더…" |
| **What** | 타이핑 중에 슬라이더를 10분으로 변경 |
| **How** | 슬라이더 펼치고 드래그 → 남은 3분이 새 비율로 재계산되어 6분이 됨 → 조용히 타이머 UI만 갱신, 확인 팝업 없음 → 계속 타이핑 |

**거치는 기능**: FN-05 (입력 중 조정) → 재계산 공식 적용

**주의점**: 타이머 재계산 공식이 직관적이지 않을 수 있음. 디자인팀에 **"슬라이더 변화 애니메이션으로 자연스럽게 표현"** 요청 필요

**감정 변화**: "지금 이 순간 멈추지 않고 이어갈 수 있구나" 연속감

## 시나리오 5 — 갑자기 그만두기

| 항목 | 내용 |
|------|------|
| **Who** | 사용자 |
| **When** | 3분 타이핑하다가 갑자기 졸림 |
| **Where** | 침대 |
| **Why** | 타이핑하다가 생각을 정리한 것만으로도 졸리기 시작함. 굳이 끝까지 기다리지 않고 싶음 |
| **What** | 그냥 폰을 내려놓거나 화면을 끔 |
| **How** | ① 폰 화면 자동 잠금 → 브라우저 탭 백그라운드 → 타이머 여전히 돌아감 → 내일 아침엔 페이지가 이미 사라진 상태로 남아 있음 ② 또는 브라우저 앱 종료 → 페이지 닫힘 → 아무것도 안 남음 |

**중요한 디자인 결정**:
- **Q1**: 백그라운드에서 타이머 계속 도는가?
- **A (제안)**: 브라우저 탭이 백그라운드여도 JavaScript setTimeout은 계속 실행됨 → 타이머 종료 시점에 화면은 사라짐. 다만 모바일 브라우저는 백그라운드 탭을 절전 모드로 돌릴 수 있어, 타이머가 약간 지연될 수 있음
- **Q2**: 사용자가 돌아왔을 때 무엇이 남아 있어야 하는가?
- **A (제안)**: 돌아왔을 때 이미 타이머 종료되었으면 퇴장 안내도 자동으로 사라진 후 새 빈 캔버스. 즉 "돌아왔을 때 항상 깨끗한 상태"

**감정 변화**: "굳이 마무리 안 해도 괜찮아" 부담 없는 느낌

**🚨 PM 결정 필요**: 위 Q1/Q2의 제안대로 가도 되는지 확인 필요. 만약 "백그라운드에서도 애니메이션이 확실히 돌아야 한다"면 기술적 제약 있음

## 시나리오에서 도출된 공통 감정 루프

```
[진입 전]          [진입 직후]         [타이핑 중]          [사라짐 순간]      [퇴장 안내 후]
 불안·피로  →  익숙·안정감  →  해방감  →  후련함+허탈  →  안도·졸음
```

이 루프 전체에서 **어떤 단계도 사용자가 "피곤하다"고 느끼면 안 됨**.
→ CX팀의 카피·톤, 디자인팀의 모션이 이 루프를 방해하지 않는 것이 핵심

## 네거티브 시나리오 (일어나면 안 되는 상황)

- 사용자가 타이머를 보고 "아 벌써 5분 됐어?" 허둥지둥 하는 느낌 → **시간 압박감 금지**
- 사라지는 순간 "내가 쓴 거 진짜 사라졌어?! 아까운데…" 후회 → **후회 유발 금지**
- 사용자가 설정이 많아서 "어떻게 쓰는 거야?" 혼란 → **학습 곡선 0에 가까워야 함**
- "사라진 메모 되돌릴 수 있어?" 질문이 FAQ에 올라오는 상황 → **복구 불가라는 메시지가 명확해야 함** (CX팀 과제)
