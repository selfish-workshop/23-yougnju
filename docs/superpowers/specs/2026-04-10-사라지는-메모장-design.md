# 사라지는 메모장 — 디자인 스펙

**작성일**: 2026-04-10
**작성자**: PM (이영주)
**상태**: 초안 (디자인팀 인계 전)
**단계**: Brainstorming 완료 → 각 팀 상세 작업 대기

---

## 0. 배경

새벽 3시, 잠들지 못하고 머릿속에 꼬리 무는 생각을 쏟아낼 곳이 없는 직장인을 위한 웹 도구. 저장도, 로그인도, 복구도 없다. 오직 "지금 이 순간의 생각"을 적고 5분 뒤 자동으로 사라지게 만들어 "비워내고 잠들기"를 돕는다.

이 스펙은 Brainstorming 단계에서 PM이 결정한 11가지 핵심 사항을 기록한 문서다. 5개 팀(기획·디자인·CX·마케팅·전략)은 이 프레임 위에서 각자의 상세 작업을 이어간다.

## 1. 서비스 정체성

| 항목 | 내용 |
|------|------|
| **이름** | 사라지는 메모장 |
| **한 줄 설명** | 적고 5분 뒤 사라지는, 불면증 직장인의 새벽 친구 |
| **핵심 약속** | 서버는 당신이 쓴 글을 1초도 저장하지 않습니다 |
| **유형** | 웹 도구 (정적 웹앱) |
| **핵심 가치** | 마음 비우기 / 스트레스 해소 / 잠에 드는 것 |

---

## 2. 타겟 사용자

**대표 페르소나**: **새벽 3시 잠 못 드는 직장인**

- 낮 동안의 스트레스, 상사와의 갈등, 내일 해야 할 일, 꼬리 무는 자책이 머릿속을 맴돌아 잠을 못 이룸
- 침대에서 폰을 만지며 SNS를 뒤지다가 더 피로해짐
- 일기장은 부담스럽고("내일 또 보면 우울할 것 같아"), 메모장은 쌓여서 싫음
- 누구에게도 말 못 할 생각을 쏟아낼 곳이 필요함
- 목적은 **"생각 정리"가 아니라 "비우고 잠드는 것"**

---

## 3. 핵심 결정 사항 (11가지)

Brainstorming 과정에서 PM이 내린 11가지 결정. 이 결정들은 각 팀의 상세 작업을 가이드한다.

| # | 항목 | 결정 | 이유 |
|---|------|------|------|
| 1 | **핵심 목적** | 마음 비우기 / 스트레스 해소 | 워킹메모리 보조나 자기표현이 아닌, 감정 해방이 1차 가치 |
| 2 | **타겟** | 새벽 3시 잠 못 드는 직장인 | 가장 구체적이고 간절한 니즈를 가진 대상 |
| 3 | **사라지는 방식** | 자동 타이머 | 새벽엔 능동적 버튼 조작도 부담 → 가만히 둬도 알아서 사라짐 |
| 4 | **타이머 정책** | 슬라이더 30초 ~ 1시간, 기본 5분 | 상황별 유연성 확보, 단 첫 진입 시 결정 부담 없도록 기본값 제시 |
| 5 | **작성 방식** | 캔버스 (자유 타이핑, 페이지 단위 타이머) | 의식의 흐름으로 길게 써도 편함, 심플한 메타포 |
| 6 | **사라지는 연출** | 잉크 번짐 → 투명화 | 편지를 물에 적시는 감성, 브랜드 시그니처 모먼트 |
| 7 | **플랫폼 우선순위** | 모바일 우선 (Mobile-First) | 새벽 3시엔 폰만 손에 있음, 터치 친화적 |
| 8 | **사라진 직후** | "이제 자러 가볼까요?" 퇴장 유도 | 연속 사용이 아니라 "잠드는 것"이 진짜 목적 |
| 9 | **로그인·데이터** | 반익명, localStorage만, 서버 저장 X | 브랜드 약속 100% + 사용자 편의(설정 기억) |
| 10 | **사운드·햅틱** | 햅틱만, BGM은 MVP 제외 | 밤엔 소리 부담, 조용한 진동 피드백이 "곁에 있어주는 느낌" |
| 11 | **MVP 범위** | Target MVP (결정된 것 전부, 유료·테마·공유는 제외) | 브랜드 경험의 골격은 온전히, 부가 기능은 검증 후 추가 |

## 4. 사용자 여정

```
[진입]
 ├─ URL 접속
 ├─ 첫 진입자: 한 줄 안내 ("적어봐요. 5분 뒤 사라집니다.") → 1회 노출 후 꺼짐
 └─ 재방문자: 바로 빈 캔버스

[작성]
 ├─ 상단에 타이머 슬라이더 (기본 5분, 지난 설정 기억)
 ├─ 가운데 캔버스, 제목 없음, 바로 본문 입력
 ├─ 첫 타이핑 순간부터 카운트다운 시작
 └─ 타이핑할 때마다 미묘한 햅틱 진동

[사라짐]
 ├─ 타이머 종료
 ├─ 잉크 번짐 애니메이션 (2~3초, 글자가 물에 젖듯 번지며 흐려짐)
 └─ 완료 순간 부드러운 햅틱 (마지막 인사)

[퇴장 유도]
 ├─ "오늘은 여기까지. 이제 자러 가볼까요?" (2~3초 노출)
 └─ 화면 탭 → 새 빈 캔버스 (연속 사용 가능하지만 강요 X)
```

## 5. 화면 구조 (모바일 우선)

```
┌─────────────────────┐
│  ⏱ 5분  ◀─────▶     │  ← 상단 슬라이더 (탭하면 펼쳐짐)
│                     │
│                     │
│  │                  │  ← 가운데 캔버스
│  │ (커서 여기에)     │     제목 없음, 바로 본문
│  │                  │
│                     │
│                     │
│  • • •              │  ← 하단 상태 (타이머 시작 전: 안내 텍스트)
└─────────────────────┘
```

**레이아웃 원칙**
- 최대 너비 480px
- 데스크탑에서는 중앙 정렬로 동일 경험 유지
- 다크 모드 기본: 배경 딥네이비 `#1a1d2e`, 글자 크림색 `#f5ebd8`
- 폰트: 손글씨 느낌 세리프 (타이핑임에도 "글쓰기"의 감성)

*구체적 컬러 팔레트·폰트·모션 수치는 디자인팀이 상세화한다.*

## 6. 기술 아키텍처

**정적 웹앱 — 서버 없음**

| 레이어 | 구성 |
|--------|------|
| **프론트엔드** | HTML + CSS + Vanilla JS (프레임워크 선택은 구현팀 재량) |
| **호스팅** | Vercel / Netlify / GitHub Pages 중 선택 (무료) |
| **백엔드** | **없음** ← 핵심 브랜드 약속 |
| **데이터베이스** | **없음** |
| **저장소** | 브라우저 localStorage (사용자 설정 한정) |

### localStorage 스키마

```js
{
  "preferredDuration": 300,    // 초 단위, 기본 300 (5분)
  "hasSeenIntro": true,        // 첫 진입 안내 노출 여부
  "hapticEnabled": true        // 햅틱 on/off
  // ⚠️ 메모 본문은 절대 저장 안 함
}
```

### 이 결정의 의미

- **GDPR·개인정보보호법 리스크 거의 0** (서버에 개인 데이터 없음)
- **초기 비용 0** (서버·DB 없음)
- **스케일 무한** (정적 파일 배포)
- **오프라인 동작 가능** (PWA 확장 여지)

## 7. 핵심 인터랙션

| 이벤트 | 반응 |
|--------|------|
| 첫 진입 | 한 줄 안내 → 1회만 노출 |
| 타이핑 시작 (첫 글자 입력) | 타이머 카운트다운 시작, 미묘한 햅틱 |
| 추가 타이핑 (이미 진행 중) | 타이머에 영향 없음 (첫 입력 기준으로 계속 카운트다운) |
| 슬라이더 조정 (입력 전) | 실시간 타이머 갱신 |
| 슬라이더 조정 (입력 중) | 남은 시간을 새 설정값 비율로 재계산, 조용히 갱신 (확인 프롬프트 없음) |
| 타이머 종료 | 잉크 번짐 2~3초 → 사라짐 → 퇴장 안내 |
| 탭하기 (사라진 후) | 새 빈 캔버스 |
| 뒤로가기 | 브라우저 기본 동작 (즉, 메모 즉시 사라짐) |

## 8. 에러·엣지 케이스

| 상황 | 동작 |
|------|------|
| 네트워크 오프라인 | 문제 없음 (서버 통신 없음) |
| 중간 새로고침 | 메모 전체 사라짐, 경고 없음 (의도된 동작) |
| localStorage 접근 불가 (프라이빗 모드 등) | 기본값으로 동작, 매번 초기화 |
| 타이핑 안 하고 대기 | 타이머 시작 안 함 (빈 캔버스면 아무 일도 없음) |
| 매우 긴 글 | 캔버스 스크롤 가능, 타이머는 캔버스 단위 유지 |
| 햅틱 미지원 기기 | 조용히 무시, 다른 기능은 정상 동작 |

## 9. MVP 범위

### ✅ 포함 (MVP)

- 캔버스 자유 타이핑
- 슬라이더 (30초 ~ 1시간, 기본 5분)
- 잉크 번짐 사라짐 애니메이션
- "이제 자러 가볼까요?" 퇴장 안내
- 햅틱 피드백
- localStorage 설정 저장
- 다크 테마
- 모바일 반응형 (데스크탑은 중앙 정렬로 대응)
- 한국어

### ❌ 제외 (향후 버전)

- BGM / 사운드
- 라이트 테마 / 테마 선택
- 공유 카드 / 스크린샷 방지
- 사용 통계
- 유료 플랜
- 다국어 (영어, 일본어)
- 이모지 반응, 스티커
- 계정·로그인·크로스 기기 동기화
- PWA (설치 가능한 앱)
- 알림 / 리마인더

**제외 항목은 "나중에 보기"일 뿐 "버림"이 아니다.** 런칭 후 사용자 반응에 따라 전략팀이 우선순위를 매긴다.

## 10. 5팀 연결

이 스펙 위에서 각 팀이 이어갈 상세 작업.

| 팀 | 이 결정이 주는 방향 | 작업 폴더 |
|----|---|---|
| **기획** (Planning) | 11가지 결정 기반 기능 명세서 작성, 사용자 시나리오 구체화, 우선순위 | `planning/` |
| **디자인** (Design) | 다크 톤 팔레트 선정, 잉크 번짐 모션 프로토타입, 슬라이더 UI, 타이포그래피 | `design/` |
| **CX** | "새벽 3시 직장인" 페르소나 심화, 온보딩 최소화 전략, FAQ, 감정·위기 상황 대응 | `cx/` |
| **마케팅** | 불면증 직장인 타겟 카피, "서버는 1초도 저장하지 않습니다" 신뢰 마케팅, SNS 감성 포스팅 | `marketing/` |
| **전략** | BGM·테마·프리미엄 설정 저장 등 유료 확장 로드맵, B2B (기업 복지) 가능성 검토 | `strategy/` |

## 11. 열린 질문 (디자인팀·CX팀에서 결정할 사항)

*PM 단계에선 결정하지 않고, 상세 작업 시 해당 팀이 탐구할 것들:*

- 잉크 번짐 애니메이션의 정확한 곡선과 속도
- 캔버스 배경에 종이 질감을 넣을지, 완전한 평면으로 갈지
- 슬라이더의 시각적 스타일 (숫자 노출 여부, 마커 간격)
- 첫 진입 안내 문구의 정확한 카피
- "자러 가볼까요?" 이후 2~3초를 어떻게 채울지 (추가 텍스트? 이미지?)
- 위기 상황(자해·우울 언급) 감지 시 대응 — CX팀 민감 이슈
- 접근성 (스크린 리더, 색약 대응)
